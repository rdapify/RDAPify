**الملف**: `src/infrastructure/http/RetryStrategy.ts`

**الميزات**:
- استراتيجيات إعادة محاولة متعددة (أسي، خطي، ثابت)
- تراجع أسي مع اهتزاز
- نمط قاطع الدائرة (حالات مغلقة/مفتوحة/نصف مفتوحة)
- إعادة محاولة قابلة للتكوين بناءً على نوع الخطأ
- إعادة محاولة قابلة للتكوين بناءً على رمز الحالة
- تتبع تلقائي للفشل

**الفوائد**:
- موثوقية محسّنة للفشل العابر
- يمنع الفشل المتتالي مع قاطع الدائرة
- يقلل الحمل على الخدمات الفاشلة
- توقيت إعادة محاولة ذكي

**واجهة برمجة التطبيقات**:
```typescript
import { RetryStrategy } from 'rdapify';

const strategy = new RetryStrategy({
  strategy: 'exponential-jitter',
  maxAttempts: 5,
  initialDelay: 1000,
  maxDelay: 30000,
  circuitBreaker: {
    enabled: true,
    threshold: 5,
    timeout: 60000,
  },
});

// التحقق من ضرورة إعادة المحاولة
const shouldRetry = strategy.shouldRetry({
  attempt: 1,
  error: new Error('ETIMEDOUT'),
  startTime: Date.now(),
});

// الحصول على التأخير للمحاولة التالية
const delay = strategy.getDelay(2); // يُرجع التأخير بالميلي ثانية

// تسجيل النجاح/الفشل
strategy.recordSuccess();
strategy.recordFailure();

// الحصول على حالة الدائرة
const state = strategy.getCircuitState(); // 'closed' | 'open' | 'half-open'
```

- التحقق من رقم الهاتف

**الفوائد**:
- دعم النطاقات الدولية
- دعم أفضل لـ IPv6
- استعلامات ASN أكثر مرونة
- التحقق الشامل من المدخلات

**واجهة برمجة التطبيقات**:
```typescript
import {
  validateIdnDomain,
  validateIpv6WithZone,
  validateAsnRange,
  idnToAscii,
  asciiToIdn,
} from 'rdapify';

// نطاقات IDN
const ascii = validateIdnDomain('مثال.السعودية');
// يُرجع: xn--mgbh0fb.xn--mgberp4a5d4ar

// IPv6 مع المنطقة
const result = validateIpv6WithZone('fe80::1%eth0');
// يُرجع: { ip: 'fe80::1', zone: 'eth0' }

// نطاق ASN
const range = validateAsnRange('AS15169-AS15200');
// يُرجع: { start: 15169, end: 15200 }

// تحويل Punycode
const punycode = idnToAscii('例え.jp');
const idn = asciiToIdn('xn--r8jz45g.jp');
```

### 4. ذاكرة التخزين المؤقت الدائمة ✅

**الملف**: `src/infrastructure/cache/PersistentCache.ts`

**الميزات**:
- تخزين قائم على الملفات وقائم على الذاكرة
- فترات حفظ تلقائية
- دعم TTL (وقت البقاء)
- فرض الحد الأقصى للحجم مع إزالة LRU
- إحصائيات ذاكرة التخزين المؤقت
- تنظيف الإدخالات منتهية الصلاحية

**الفوائد**:
- تبقى ذاكرة التخزين المؤقت بعد إعادة تشغيل التطبيق
- بدء تشغيل أسرع مع البيانات المحملة مسبقًا
- تقليل استدعاءات API
- أداء أفضل

**واجهة برمجة التطبيقات**:
```typescript
import { PersistentCache } from 'rdapify';

const cache = new PersistentCache({
  storage: 'file',
  path: './cache/rdap-cache.json',
  ttl: 3600000, // ساعة واحدة
  maxSize: 1000,
  autoSave: true,
  saveInterval: 60000, // دقيقة واحدة
});

// تعيين القيمة
await cache.set('key', 'value', 7200000); // TTL مخصص: ساعتان

// الحصول على القيمة
const value = await cache.get('key');

// التحقق من الوجود
const exists = await cache.has('key');

// حذف القيمة
await cache.delete('key');

// الحصول على الإحصائيات
const stats = await cache.getStats();
console.log(`الحجم: ${stats.size}/${stats.maxSize}`);
console.log(`النجاحات: ${stats.hits}`);

// تنظيف الإدخالات منتهية الصلاحية
const removed = await cache.cleanup();

// تدمير ذاكرة التخزين المؤقت
cache.destroy();
```

- **تحسين الموارد**: تزامن محكوم

- **ASN مرن**: استعلامات النطاق للعمليات الجماعية

- **بدء تشغيل أسرع**: بيانات ذاكرة التخزين المؤقت المحملة مسبقًا

- **أداء أفضل**: لا عقوبة بدء تشغيل بارد
