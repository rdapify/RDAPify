**RDAPify** يوحد استعلامات RDAP عبر جميع السجلات العالمية (Verisign, ARIN, RIPE, APNIC, LACNIC) مع حماية أمنية قوية وأداء استثنائي وتجربة مطور متكاملة. هذا ليس مجرد عميل RDAP آخر — إنه منصة كاملة لمعالجة بيانات التسجيل بشكل آمن.

> **ملاحظة:** هذا المشروع يلغي الحاجة لبروتوكول WHOIS التقليدي، مع الحفاظ على التوافق العكسي عند الحاجة.

## 🚀 لماذا RDAPify؟

استعلامات RDAP المباشرة معقدة — كل سجل يستخدم تنسيقات مختلفة، وحدود معدل، وإجراءات أمنية. بدلاً من إعادة اختراع العجلة لكل مشروع:

```diff
- صيانة تطبيقات WHOIS/RDAP متعددة
- معالجة اختلافات السجلات يدوياً
- قلق مستمر من ثغرات SSRF
- أداء غير متوقع بدون تخزين مؤقت
+ حل موحد واحد، مختبر بدقة، جاهز للإنتاج
```

RDAPify يعالج هذه التحديات بذكاء:

- ✅ **توحيد البيانات**: استجابة متسقة بغض النظر عن السجل المصدر
- ✅ **حماية SSRF**: منع الهجمات على البنية التحتية الداخلية
- ✅ **أداء استثنائي**: تخزين مؤقت ذكي، معالجة متوازية، تحسين الذاكرة
- ✅ **توافق واسع**: يعمل على Node.js, Bun, Deno, Cloudflare Workers
- ✅ **جاهز لـ GDPR**: أدوات مدمجة لإخفاء البيانات الشخصية تلقائياً

## ⚡ ابدأ في 30 ثانية

```typescript
import { RDAPClient } from 'rdapify';

// إنشاء عميل آمن مع إعدادات محسّنة
const client = new RDAPClient({
  cache: true, // تخزين مؤقت تلقائي (TTL ساعة واحدة)
  privacy: true, // إخفاء المعلومات الشخصية تلقائياً
  retry: {
    // إعادة محاولة ذكية للفشل المؤقت
    maxAttempts: 3,
    backoff: 'exponential',
  },
});

// استعلام عن نطاق
const result = await client.domain('example.com');

console.log({
  domain: result.query,
  registrar: result.registrar?.name,
  status: result.status,
  nameservers: result.nameservers,
  created: result.events.find((e) => e.type === 'created')?.date,
  expires: result.events.find((e) => e.type === 'expiration')?.date,
});
```

## 🌟 الميزات الأساسية

### 🔒 أمان مؤسسي

- **حماية SSRF مدمجة**: منع الاستعلامات لعناوين IP الداخلية أو النطاقات الخطرة
- **التحقق من الشهادات**: رفض الاتصالات غير الآمنة لخوادم RDAP
- **تحديد المعدل**: منع حظر الخدمة بسبب الطلبات المفرطة
- **معالجة بيانات آمنة**: إخفاء البيانات الشخصية وفقاً لمتطلبات GDPR/CCPA
- **سجل تدقيق كامل**: تتبع جميع العمليات الحرجة لأغراض الامتثال

### ⚡ أداء استثنائي

- **تخزين مؤقت متقدم**: يدعم التخزين في الذاكرة، Redis، والتخزين الموزع جغرافياً
- **معالجة متوازية**: معالجة 1000+ نطاق في ثوانٍ
- **اكتشاف السجل**: اكتشاف تلقائي عبر IANA Bootstrap للعثور على السجل الصحيح
- **وضع عدم الاتصال**: العمل مع البيانات المخزنة مؤقتاً أثناء انقطاع الشبكة
- **قوائم الأولوية**: ضمان معالجة الطلبات الحرجة أولاً

### 🧩 تكامل سلس

- **دعم TypeScript كامل**: مكتوب بقوة مع توثيق مدمج
- **دعم بيئات متعددة**: يعمل على Node.js (16+), Bun, Deno, Cloudflare Workers
- **CLI تفاعلي**: للاستعلامات السريعة والاختبار
- **ساحة تجربة ويب**: جرب المكتبة مباشرة في متصفحك
- **قوالب جاهزة**: لـ AWS Lambda, Azure Functions, Kubernetes، والمزيد

## 🛡️ الأمان كمبدأ أساسي

نحن لا نعامل الأمان كميزة إضافية — إنه أساسي في تصميمنا. RDAPify يحمي تطبيقاتك من:

| التهديد        | آلية الحماية                                       | الأهمية |
| -------------- | -------------------------------------------------- | ------- |
| SSRF           | التحقق من النطاق، حظر IPs الداخلية                 | 🔴 حرج  |
| DoS            | تحديد المعدل، مهلات زمنية                          | 🟠 مهم  |
| تسريب البيانات | إخفاء البيانات الشخصية، عدم تخزين الاستجابات الخام | 🔴 حرج  |
| MitM           | HTTPS إلزامي، التحقق من الشهادات                   | 🟠 مهم  |
| حقن البيانات   | التحقق من المخطط، تحليل صارم                       | 🟠 مهم  |

اقرأ [ورقة الأمان البيضاء](security/whitepaper.md) للتفاصيل التقنية العميقة والسيناريوهات المتقدمة.
